Developer Journal

Technologies Used

  • HTML
  • CSS
  • Javascript

General Information

This developer journal provides a more organized environment for developers to keep all of their thoughts such as code snippets, github issues, design notes, meeting notes, and freeform notes. The journal can store multiple projects and each project can contain an unlimited amount of different notes in it in order for developers to separate their notes based on each project. There are export and import options that allow developers to save their journal and either move it to another device or to send to another developer so they can upload a copy of their journal. Finally, there is also a chatbot feature where users can use a chatbot API and enter a query and receive an appropriate response.

General Struggles and Solutions

Me and a group of developers were tasked to create a journal specifically for developers. We decided to start by brainstorming a bunch of features that developers may want in a journal such as a note that allows for markdown and also code snippets. In addition, we thought it would be helpful to include design and meeting notes in order to organize the Figma designs that their currently trying to recreate on the front end and also organize thoughts, comments and feedback from any meetings they may have. The main struggle we had was figuring out a way to export and import journal data. This would allow developers to save and export their projects notes from their journal and import it into a journal on another device. This would provide users with more flexibility when it comes to data transfer and would allow developers to collaborate by sharing each other's notes. After some thought, we decided that we could serialize the data into json files and export that json file for users to download and in order to transfer that data, users could just import that json file into our website, in which we'd deserialize the data, then import all of the data into the website for them to use!
