Haunted Mansion

Technologies Used

  • C++

General Information

Haunted Mansion is a text-based game that takes users through a story-driven adventure where the protagonist goes through an old, abandoned building. There are a total of 6 different endings that the users can receive based on their actions throughout the game. Try it out to find all the endings!

General Struggles and Solutions

This was my first time creating my own coding project. I'd just finished taking an Object-Oriented Programming class and had learn about the basic data structures such as arrays and linked lists. For my first project, I wanted to create a simple game. I had always loved adventure games and visual novels, where players could read a bunch of dialogue and make choices to conclude to a certain ending based on those decisions. A choices-matter type of game. I had a general story plot in mind but I wanted players to be able to interact with locations and use items to unlock certain areas. Therefore, I needed to plan out a way to keep track of the player's items as they play. I originally thought about using an array to store the items but it wasn't very efficient when it came to removing and finding a specific item. A similar case for linked lists, in order to figure out whether a player had a specific item, we'd have to iterate through the entire linked list. Since there were operations that run in O(n) time when using arrays and linked list, I needed to find something faster. This is when I started doing some research on ways to make adding items, removing items, and checking an item's existence more efficient. This is when I came across and decided to use a hashmap, which checks all of the boxes. By using a hashmap, I'm able to check an item's existence by checking its corresponding key value. In addition, adding and removing these items are fairly easy as well, all running in O(1) time.
